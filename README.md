# pipery.dev

This app is built with Hugo and deployed to GitHub Pages through GitHub
Actions.

## How it works

- The site is generated by Hugo into `public/`.
- The GitHub Actions workflow at
  `.github/workflows/release-static-page.yml` runs on pushes to `main` and on
  manual dispatch.
- The workflow uses `actions/configure-pages` and deploys the generated
  `public/` directory with `actions/deploy-pages`.
- During CI, Hugo is built with the actual GitHub Pages base URL so the site
  works correctly on both custom domains and `*.github.io` URLs.

## GitHub Pages setup

In your repository settings, configure GitHub Pages to deploy from:

- Source: `GitHub Actions`

If a custom domain is intended, set it in GitHub Pages settings and add a
`CNAME` file if needed. Without that, the canonical Pages URL will be the
`*.github.io` address returned by GitHub Actions.
